## 📊 주요 기능

### 1. 📝 RTD Test (Real-Time Decision Test)

| Step | 기능 명 | BackEnd 요청 여부 | 상세 내용 |
| :--- | :--- | :--- | :--- |
| **1** | 사업부 선택 UI | - | |
| **2** | 개발라인 선택 | O | 사업부 선택 결과에 따라 List 갱신 |
| **3** | Test 대상 Rule 선택 | O | 개발라인 기준으로 List 갱신 |
| **4** | Old/New Rule 버전 확인 | O | 선택한 Rule 기준으로 정보 표시 |
| **5-1** | 타겟 라인 선택 | O | 사업부에 따른 타겟라인 List 제공. **전체선택 버튼 필수** |
| **5-2** | Test.jar 수행 요청 | O | 타겟 라인 기준으로 요청. **라인별 병렬 진행** |
| **5-3** | 진행 상태 표시 | O | 진행 중인 Test가 있는 경우 **Test 버튼 비활성화** |
| **6-1** | 결과 다운로드 | O | Test 종료 시 라인별 테스트 결과 다운로드 버튼 활성화 (Raw Data) |
| **6-2** | 종합 결과 생성 | O | 전후 변경점 Text 입력 후 종합 test 결과 생성/다운로드 버튼 활성화 |
| **7** | 결과 상시 조회 | O | 마지막 요청한 결과는 마이페이지에서 조회 가능 |
| **세션 관리** | 진행 과정 저장/초기화 | O | 이전 진행 과정(Step)이 User 기준으로 계속 저장되어야 함. (초기화 버튼 제공) |
| **Flow** | Step 제어 | 이전 Step 결과 변경 시, 다음 Step부터 **다시 시작** 가능하도록 UI 제어 |

## 👑 Admin Menu (특정 유저에게 Admin 권한 부여 후 접근 가능)

| 구분 | 기능 명 | BackEnd 요청 여부 | 상세 내용 |
| :--- | :--- | :--- | :--- |
| **User 관리** | 사용자 관리 | O | 유저 가입 승인, 유저 삭제, Admin 권한 부여/회수 기능 |
| **RTD 관리** | RTD 설정 관리 | O | 사업부 별 라인 및 home dir 경로 관리 (추가/변경/삭제) |
| **ezDFS 관리** | ezDFS 설정 관리 | O | 타겟 서버 별 dir 경로 관리 (추가/변경/삭제) |
